// how it works

LLM for the new.
IMPRINT for the known.

Two components. One learns from experience and caches structured plans. The other controls your phone via ADB and UIAutomator. They work together — and over time the LLM barely fires at all.

imprint
Intent arrives → extract template + params
"message mom hey" → template + {contact} + {message}
imprint
TF-IDF similarity search against SQLite cache
No vector DB. No embeddings API. Pure local math.
llm (only on miss)
Cerebras GPT-OSS-120B plans abstract steps
Element-based targeting, not raw coordinates.
imprint + wrangle
Execute via ADB + UIAutomator. Verify each step.
Mid-task replan on failure. Per-step retry.
stored
Plan cached. Next similar intent: 0 tokens.
Requires N confirmations before trusted. Auto-evicts on repeated failure.
// architecture
OpenClaw (chat / dispatcher) │ natural language intent ▼ imprint.py ←── SQLite (~/.imprint/memory.db) │ JSON action steps ▼ wrangle.py ←── ADB → Android device │ JSON results ▼ imprint.py → summary / stats
// features

no hallucinations.
no subscriptions.

01
parameterized skill templates
Plans use {slot} placeholders. "Message Mom" and "Message Dad" share one cached template. Variable content is extracted and resolved at runtime.
02
element-based targeting
Steps target UI elements by text, resource-id, or content-desc — not raw coordinates. Works even if the screen layout shifts slightly.
03
confirmation threshold
New plans start as PENDING. Only promoted to TRUSTED after N successful real-world executions. You control how paranoid it is.
04
mid-task replan
If a step fails mid-execution, IMPRINT calls the LLM with the current screen state and remaining steps to recover — without starting over.
05
destructive action guard
Delete, uninstall, purchase, reset — all blocked by default. Requires explicit --confirmed flag or interactive y/N prompt. Not bypassable by the LLM.
06
offline queue
Queue tasks with --queue when ADB is unavailable. Drain the backlog with imprint.py flush on reconnect. Nothing dropped.

// quickstart

running in under 10 minutes.

# In Termux on your Android device
pkg update && pkg install git python openssh

pip install requests

git clone https://github.com/inoahaipro/wrangle-imprint.git
cd wrangle-imprint
mkdir -p ~/.imprint

# Set your Cerebras key
export CEREBRAS_KEY=csk-your-key-here
# Install ADBKeyboard APK on your phone, then:
adb shell ime set com.android.adbkeyboard/.AdbIME

# Verify ADB + everything else
python wrangle.py check
python imprint.py check

# Expected output:
── wrangle setup check ──
✅ ADB: localhost:34371 connected
✅ ADBKeyboard: active
✅ CEREBRAS_KEY: set
# First run — cache miss, LLM plans it
python imprint.py ask "open youtube and search for lo-fi hip hop"

# Same task again — cache hit, 0 tokens
python imprint.py ask "open youtube and search for drum and bass"

# Destructive actions need explicit confirmation
python imprint.py ask "delete that message" --confirmed

# Inspect what's stored
python imprint.py list
python imprint.py stats
# Copy and fill in .env.example
cp env.example .env

# Key settings
CEREBRAS_KEY=csk-...  # required
IMPRINT_THRESHOLD=0.72  # cache match strictness
IMPRINT_CONFIRM=2  # runs before "trusted"
IMPRINT_MAX_STEPS=20  # safety cap
IMPRINT_DEBUG=1  # verbose logging
DEVICE_WIDTH=1080  # your screen res
DEVICE_HEIGHT=2340  # S22+ default

source .env
