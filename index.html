// how it works

Firewall first. LLM as a fallback.

Token Firewall speaks the OpenAI API so your client doesn’t have to change. Under the hood it routes requests through an intent engine, a two‑tier cache (static packs + learned entries), and platform hands that can actually do things on devices. Only when there’s no good answer or plan does it call your underlying LLM.

client (OpenClaw / CLI / web UI) │ ▼ server.py (FastAPI · SSE · /v1/chat/completions) │ ▼ FirewallRouter ├── IntentEngine · classify prompt · split tasks ├── KnowledgeStore · packs + learned SQLite cache ├── PlatformHands · execute device actions │ ├── AndroidHands (Termux API + ADB) │ ├── DesktopHands (shell) │ └── IOSHands (future) └── LLMAdapter · fallback chain of models
// quickstart

running locally in a few commands.

Token Firewall is designed to run close to your agents: Termux on Android, or a normal shell on desktop. Point your OpenAI‑compatible client at it and let it worry about caching, routing, and device actions. 

# In Termux
pkg update && pkg install python android-tools termux-api
pip install fastapi "uvicorn[standard]"

# Clone and run
cd ~
git clone https://github.com/inoahaipro/token_firewall.git
cd token_firewall

# Configure your upstream model
nano ~/.token-firewall.env
#   TF_LLM_BASE_URL=...
#   TF_LLM_API_KEY=...
#   TF_LLM_MODEL=...

bash start.sh
# → serves /v1/chat/completions and /ui on http://127.0.0.1:8000
# On Linux/macOS/WSL
python -m venv venv
source venv/bin/activate
pip install fastapi "uvicorn[standard]"

cd ~/code
git clone https://github.com/inoahaipro/token_firewall.git
cd token_firewall

export TF_LLM_BASE_URL=https://api.openai.com/v1
export TF_LLM_API_KEY=sk-...
export TF_LLM_MODEL=gpt-4.1-mini

python server.py
# UI: http://127.0.0.1:8000/ui
# Add Token Firewall as a provider in ~/.openclaw/openclaw.json
"models": {
  "providers": {
    "token-firewall": {
      "baseUrl": "http://127.0.0.1:8000/v1",
      "apiKey": "none",
      "api": "openai-completions",
      "models": [
        { "id": "firewall", "name": "Token Firewall", "input": ["text"] }
      ]
    }
  }
}

# Then pick it as your model in OpenClaw and chat as usual.
# Cache hits will stay local, misses will go out to your LLM.
